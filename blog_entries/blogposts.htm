Blog Posts, Viewing Files, and HTML Encoding

This is a blog post about making blog posts.

Here are some important points:

  • Blog posts should be placed in the directory ~/blog_entries.
  • Blog posts should be snippets of HTML that are placed in files with extension .htm
  • The snippets of HTML will be gathered by algorithms into a composite file that will be served to the client. Because of this, it is important that the following tags not be in the file:
    • <!DOCTYPE html>
    • <html>
    • <head>
    • <body>
    The file really should consist only of a snippet.

Blog posts may contain links and often should contain links to the content being discussed in the post. As an example, here is a link to view the source code of this blog post:

View: ~/blog_entries/blogposts.htm

It may sometimes be important to show HTML source code directly in a blog post. To do this, one must be clever. If you just drop the HTML source code in, the browser will process it just like any other HTML and you will see the result of the code and not its source.

To view HTML source code, you must do two steps:

  • The HTML source must be HTML encoded so that its angle brackets and other special characters are represented in a way that will not trigger the normal browser transformations.
  • The HTML encoded text must be inserted in a block <pre></pre>. This is a pre-formatted text block that will preserve the spacing and the line feeds correctly.

You do not want to do HTML encoding by hand. Instead, use the following tool:

Text to HTML

To use this tool, put the HTML source in the text box and then click the HTML Encode Text button. Then select the converted source and copy/paste it into a <pre></pre> block in your file.

Here is a summary of the conversions done by the HTML encoding process:

  • The character < converts to &lt;
  • The character > converts to &gt;
  • The character & converts to &amp;
  • The character " converts to &quot;
